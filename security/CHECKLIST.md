# Security Checklist — For AI Agents

Run the relevant section before each action type. Every checkbox must be addressed.

---

## Before Starting Any Project

```
[ ] All credentials provided in ACCESS.md — none hardcoded in any file
[ ] ACCESS.md is gitignored before first commit
[ ] .gitignore exists with: .env, .env.local, *.tfvars, *.pem, *.key, service-account*.json
[ ] GitHub branch protection is configured — no direct push to main
[ ] GitHub Secrets are set (not env vars) for: AWS keys, Cloudflare token
```

---

## Before Implementing Any Endpoint

```
[ ] Zod schema created for request body (POST, PUT, PATCH)
[ ] Zod schema created for query params (GET with filters/pagination)
[ ] Auth middleware applied to every non-public route
[ ] Role check applied to admin-only routes
[ ] Rate limiting: generic limit on all routes, strict limit on /auth/* routes
[ ] SQL queries use parameterized values ($1, $2 — never string interpolation)
[ ] File uploads: MIME type validation, size limit, stored in S3 (not served from API)
```

---

## Before Committing Any Code

```
[ ] No hardcoded secrets, tokens, API keys in any file
[ ] No .env files added to git
[ ] console.log / print statements don't log sensitive fields (passwords, tokens)
[ ] Error responses don't expose stack traces or internal paths
[ ] All acceptance criteria from plan are met
[ ] All verification commands passed (with actual output, not "it should work")
[ ] npm audit / pip audit run — no high/critical vulnerabilities
[ ] No TODO comments that mask unresolved security concerns
```

---

## Before Deploying Infrastructure

```
[ ] tofu plan reviewed — no unexpected resource deletions
[ ] All secrets are generated by OpenTofu random resources (not hardcoded in tfvars)
[ ] RDS deletion_protection = true
[ ] RDS not in public subnet — only accessible from ECS security group
[ ] S3 buckets not public by default (bucket policy explicit for what is public)
[ ] ECS task definition uses secrets field (not environment) for sensitive values
[ ] ALB has HTTPS listener — HTTP redirects to HTTPS
[ ] Security groups follow least-privilege (only required ports open)
[ ] CloudFront configured with HTTPS only, TLS 1.2 minimum
```

---

## Before Going Live (Pre-Production Checklist)

```
[ ] HTTPS enforced — HTTP returns 301 redirect to HTTPS
[ ] HSTS header configured (via helmet or CloudFront)
[ ] CORS configured with explicit origin allowlist — no wildcard *
[ ] Rate limiting active on all public routes
[ ] Error messages are generic to client ("Internal server error" not stack traces)
[ ] JWT secret is at least 64 chars (use tofu random_password with length=64)
[ ] JWT access token TTL ≤ 15 minutes
[ ] Refresh tokens stored in DB (revocable on logout)
[ ] Database backups enabled (RDS backup_retention_period ≥ 7)
[ ] CloudWatch logs configured (ECS task log configuration set)
[ ] Health check endpoint accessible — returns 200
[ ] Monitoring alert on 5xx error rate spike (CloudWatch alarm)
[ ] Unused AWS services are NOT deployed (principle of least infrastructure)
```

---

## Authentication Security

```
[ ] Passwords hashed with bcrypt (cost factor 12+) — never MD5/SHA1
[ ] Login endpoint: same error message for wrong email AND wrong password
[ ] Login endpoint: rate limited to 10 attempts per 15 minutes
[ ] Register endpoint: checks for existing email before hashing (efficient) but same response either way
[ ] JWT payload: only id and role — no PII (no email, name, etc.)
[ ] JWT: signed, not encrypted — payload is base64 decodable — verify nothing sensitive is in it
[ ] Logout: deletes refresh token from DB (not just removes from client)
[ ] Password reset: time-limited token (15 minutes), single-use, stored hash only
```

---

## SQL Security

```
[ ] ALL queries use parameterized values — grep for string template literals in SQL
[ ] UUIDs validated before use in queries (prevent UUID injection)
[ ] LIMIT clause on all SELECT queries that could return large sets
[ ] No raw SQL from user input — user provides filter values, not SQL fragments
[ ] Database user has minimal permissions: SELECT, INSERT, UPDATE, DELETE on app tables only
[ ] No schema operations (CREATE, DROP, ALTER) from application code
```

## Dependency Security

```bash
# Run before first commit and before going live
npm audit --audit-level=moderate
# If vulnerabilities found:
npm audit fix
# If npm audit fix can't resolve:
# - Check if vulnerability is in a dev dependency (lower risk)
# - Check if vulnerability affects your usage pattern
# - Document in ADR if you're accepting a known risk

# For Python
pip audit
```

---

## Infrastructure Security

```
[ ] VPC has private subnets — RDS and ECS tasks are not public
[ ] Bastion host or VPN required for direct DB access (no public DB access)
[ ] ECS task role: only permissions needed by the application
[ ] No wildcard IAM policies ("*" on any resource)
[ ] ECR image scanning enabled
[ ] CloudTrail enabled for audit logging
[ ] No IAM access keys stored in code or ECS environment variables (use roles)
```

## OWASP Top 10 — Quick Check

| Threat | Check |
|---|---|
| A01: Broken Access Control | Auth on every protected route; role checks on admin routes |
| A02: Cryptographic Failures | Passwords hashed; secrets in Secrets Manager; HTTPS only |
| A03: Injection | Parameterized SQL; Zod validation on all inputs |
| A04: Insecure Design | ADR for any deviations; checkpoints before major decisions |
| A05: Security Misconfiguration | Helmet headers; CORS allowlist; no default credentials |
| A06: Vulnerable Components | npm audit / pip audit before launch |
| A07: Auth Failures | bcrypt cost 12; JWT short TTL; refresh token rotation |
| A08: Software Integrity | GitHub branch protection; signed commits optional |
| A09: Logging Failures | CloudWatch logs; error logging in error handler |
| A10: SSRF | Validate URLs before fetch; allowlist external domains |
